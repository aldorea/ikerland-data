**5. Cognito Authentication Sequence Diagram (### Authentication Flow):** Per D-03 and D-13, create a Mermaid `sequenceDiagram` showing the full Cognito Hosted UI PKCE flow:
```
participant Browser
participant Cognito as Cognito User Pools
participant APIGW as API Gateway HTTP API
participant Lambda as Lambda Handler
participant Storage as Timestream / DynamoDB / Aurora

Browser->>Cognito: Redirect to Hosted UI (authorize endpoint)
Cognito->>Browser: Login form (username + password)
Browser->>Cognito: Submit credentials
Cognito->>Browser: Authorization code (PKCE)
Browser->>Cognito: Exchange code for tokens (token endpoint)
Cognito->>Browser: id_token + access_token (JWT, RS256)
Browser->>APIGW: GET /devices (Authorization: Bearer {access_token})
APIGW->>Cognito: Validate JWT signature (JWKS endpoint)
Cognito-->>APIGW: Valid
APIGW->>Lambda: Invoke with requestContext.authorizer.claims
Lambda->>Storage: Query via VPC endpoint
Storage-->>Lambda: Result
Lambda-->>APIGW: HTTP 200 JSON response
APIGW-->>Browser: JSON response
```
Include a prose note below the diagram: "API Gateway's built-in JWT authorizer eliminates the need for a custom Lambda authorizer. Auth is enforced at the gateway level — a misconfigured Lambda handler cannot accidentally serve unauthenticated requests." Per RESEARCH.md anti-patterns.

**6. Comparison Table 1: API Front Door (### API Front Door Comparison):** Per D-04, create the table:

| Criterion | API Gateway HTTP API v2 | API Gateway REST API v1 | AWS App Runner |
|-----------|------------------------|------------------------|----------------|
| Cost | $1.00/M requests | $3.50/M requests | ~$6-7/month base + compute |
| JWT Authorizer | Native (built-in) | Lambda authorizer required | Custom middleware |
| VPC Integration | VPC Link | VPC Link | VPC Connector |
| Usage Plans / API Keys | Not supported | Supported | Not supported |
| Latency | Lower (~10ms overhead) | Higher (~30ms overhead) | Container cold start |
| WebSocket | Not supported | Supported | Not supported |
| **Recommendation** | **Selected** — 70% cheaper, native JWT, sufficient features | Use only if API keys or usage plans required | Use for container-based APIs with steady high-concurrency traffic |

**7. Comparison Table 2: Auth Provider (### Auth Provider Comparison):** Per D-14, create the table:

| Criterion | Cognito User Pools | IAM Identity Center | Self-managed IdP (e.g., Keycloak) |
|-----------|-------------------|--------------------|------------------------------------|
| Use Case | Application user authentication (operators, admins) | Workforce/employee access to AWS console | Full control over auth flows |
| JWT Issuance | Native (id_token, access_token) | SAML 2.0 assertions (not JWT directly) | Custom (depends on implementation) |
| API Gateway Integration | Native JWT authorizer — zero code | Requires Lambda authorizer to translate SAML | Requires Lambda authorizer |
| MFA | Built-in (SMS, TOTP) | Built-in | Self-managed |
| Managed | Fully managed, zero ops | Fully managed | Self-hosted infrastructure |
| Cost | Free up to 50K MAU | Free for AWS accounts | Infrastructure + operations cost |
| **Recommendation** | **Selected** — purpose-built for app users, native API GW integration | Use as SAML IdP federated with Cognito for corporate SSO | Avoid — operational overhead contradicts serverless architecture |

**8. Design Notes (### Design Notes):** Include:
- RDS Proxy is MANDATORY for Lambda → Aurora (connection exhaustion is #1 failure mode — Lambda scales to hundreds of concurrent executions, each opening a new DB connection; Aurora max_connections ~90 at 0.5 ACU). Cross-reference doc 05 storage layer.
- Lambda arm64 (Graviton2) runtime: 20% cheaper, 19% better performance. Python 3.12 runtime.
- Lambda timeout: 29 seconds max for API Gateway integration (API Gateway has a 30-second hard limit).
- Cold start mitigation: Provisioned Concurrency optional for latency-critical endpoints; not needed at this scale.
- Cognito Identity Pools documented as OPTIONAL for direct frontend-to-AWS-service access (e.g., browser → Timestream). Primary pattern is API-mediated access through Lambda for security and auditability. Per D-03.
- Cross-references: "See 01-security-foundation.md for VPC topology and IAM roles. See 05-storage-layer.md for Timestream, DynamoDB, and Aurora configuration."
